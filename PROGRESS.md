# プロジェクト進捗サマリー

このドキュメントは、議論支援システム "Trajectory" 開発プロジェクトにおいて、これまでにセットアップ・作成されたファイルやディレクトリの概要をまとめたものです。

## 作成されたファイル・ディレクトリ一覧

### 1. 開発環境

- **`.venv/`**
  - **役割:** Pythonの仮想環境です。このプロジェクトで使用するライブラリ（パッケージ）を、お使いのコンピュータの他のPython環境から隔離し、クリーンな状態に保ちます。

- **`requirements.txt`**
  - **役割:** プロジェクトが依存するPythonライブラリのリストです。このファイルがあることで、他の環境でも `pip install -r requirements.txt` コマンド一つで同じ開発環境を再現できます。
  - **内容:** `openai`, `numpy`, `streamlit` など、開発に必要なライブラリが記述されています。

- **`.env`**
  - **役割:** OpenAIのAPIキーのような、公開すべきでない秘密情報を保存するためのファイルです。プログラムはこのファイルからキーを読み込むため、コード内に直接書き込む必要がなく、安全です。
  - **注意:** このファイルは `.gitignore` によってGitの管理対象から除外されています。

- **`.gitignore`**
  - **役割:** Gitバージョン管理システムに対して、追跡（コミット）してほしくないファイルやディレクトリを指定するためのファイルです。`.venv` や `.env` などを指定することで、不要なファイルや秘密情報がリポジトリに含まれるのを防ぎます。

### 2. ソースコードとデータ

- **`src/`**
  - **役割:** プロジェクトのメインとなるPythonソースコードを格納するディレクトリです。

- **`src/drift_detector.py`**
  - **役割:** 「脱線検知」のコアロジックを実装するスクリプトです。現在は、模擬データを読み込み、テキストをベクトル化して類似度を計算する初期機能が実装されています。

- **`data/`**
  - **役割:** プログラムが使用するデータファイルを格納するディレクトリです。

- **`data/mock_discussion_01.json`**
  - **役割:** 脱線検知ロジックの動作テストに使用する、模擬的な議論のログデータです。「議論のゴール」と「発言リスト（脱線かどうかのラベル付き）」が含まれています。

### 3. ドキュメント

- **`preStudy002.md`**
  - **役割:** 元々の開発計画書です。プロジェクトの目的、機能、技術スタックなどが定義されています。

- **`GEMINI.md`**
  - **役割:** AIアシスタント（私）が開発を進める上での思考プロセスやタスク計画を記録しているファイルです。

- **`PROGRESS.md`** (このファイル)
  - **役割:** これまでに作成したファイルとプロジェクトの全体像をまとめた、進捗サマリーです。
